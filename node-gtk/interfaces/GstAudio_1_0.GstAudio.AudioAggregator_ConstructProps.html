Optional ignoreInactivePads

ignoreInactivePads?: boolean
  • Defined in GstAudio-1.0.d.ts:1513

Don't wait for inactive pads when live. An inactive pad is a pad that hasn't yet received a buffer, but that has been waited on at least once.

The purpose of this property is to avoid aggregating on timeout when new pads are requested in advance of receiving data flow, for example the user may decide to connect it later, but wants to configure it already.

Optional minUpstreamLatency

minUpstreamLatency?: number

Inherited from Aggregator_ConstructProps.minUpstreamLatency

  • Defined in GstBase-1.0.d.ts:851

Force minimum upstream latency (in nanoseconds). When sources with a higher latency are expected to be plugged in dynamically after the aggregator has started playing, this allows overriding the minimum latency reported by the initial source(s). This is only taken into account when larger than the actually reported minimum latency.
